# Security Policy

## Supported versions

Security fixes are applied to the latest main branch.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Instead:

- Describe the issue privately to the maintainers.
- Include a clear reproduction path when possible.
- Avoid sharing sensitive details publicly until a fix is available.

## What to include

- Affected version or commit
- Impact summary
- Reproduction steps
- Any relevant logs or stack traces

## Scope

This project is a TypeScript AST patch engine. Security reports should focus on dependency issues, unsafe parsing behavior, data handling, and unexpected code execution paths.
